### BEM

BEM (Block Element Modifier) – методология именования классов в HTML/CSS, чтобы код был понятным, модульным и удобным для поддержки. 

`блок__элемент--модификатор`

Основные правила:

1. Блок - независимый элемент (menu, button, card).
2. Элемент - часть блока, не имеет смысл вне него (блок__элемент). menu__item, button__text
3. Модификатор - вариация состояния блока/элемента (блок/элемент--модификатор). button--primary, menu__item--active

Имена пишут латиницей, в нижнем регистре, слова разделяют дефисом: user-card, user-card__title

```HTML
<div class='card'>
  <div class='card__title'>Заголовок карточки</div>
  <div class='card__text'>Текст</div>
  <button class='card__button card__button--primary'>Действие</button>
</div>
```

Примеры:

1. Блок 'Форма поиска' `search-form`
   1. Элемент ввода `search-from__input`
   2. Кнопка `search-form__button`
   3. Мод в фокусе `search-form__input--focued`
2. Блок 'меню' `menu`
   1. Пункт `menu__item`
   2. Активный `menu__item menu__item--active`
   3. Выпадающий `menu--dropdown`

Нюансы:

1. Нельзя делать элемент элемент `card__title__text`
2. Мод всегда идёт вместе с базовым классом `button button--primary`


- vh (viewport height) - относительная единица измерения в CSS, равная 1% от высоты области просмотра (viewport), то есть от высоты окна браузера. 100vh - полная высота видимой области экрана
- background-size управляет тем, как фоновое изображение масштабируется внутри элемента.

  ![background-size | CSS-Tricks](https://encrypted-tbn0.gstatic.com/images?q=tbn:ANd9GcRFlsYjAa9Ba9KJkpcnxUXSt5MZbKpIpxPYug&s)

  - cover - масштабирует изображение так, что оно полностью заполнило блок, сохраняя пропорции. При этом часть картинки может обрезаться по краям. Использовать на первом экране лендинга (картинка на всю ширину и высоту), карточки товаров/статей (где важно заполнить фон, а края не критичны), декоративные фоны (потеря краёв не портит смысл). Без `background-position` может съесть важную часть картинки, почти всегда с `cover` ставят `center`.
  - contain - масштабирует изображение так, чтобы оно целиком поместилось в блок, сохраняя пропорции. В результате могут оставаться пустые области (по бокам или сверху/снизу). Использовать при логотипах, иконках, схемах, карточках с иллюстрациями, адаптивных блоках
  - display: inline-block - гибридный подход, когда элемент ведёт себя как строчный (стоит в потоке текста, не переносит строку сам по себе), но при этом к нему можно применять блочные свойства – width, height, margin, padding.


# Webkit

WebKit - движок рендеринга, который изначально развивала Apple и который лежит в основе Safari. Ранее Chrome аналогично использовал WebKit, потом ушёл на форк Blink, но из-за общей истории многие свойства в Chrome/Edge/Safari до сих пор требуют префикс -webkit-.

```CSS
background: linear-gradient(90deg, rgba(17, 68, 85, 1) 80px, rgba(255, 255, 255, 1) 80px);
-webkit-background-clip: text; /* Обрезает фон по форме текста (фон виден только внутри букв)) */
-webkit-text-fill-color: transparent; /* Прозрачный цвет букв (виден фон)) */
```


# vertical-align

vertical-align - CSS-свойство, которое задаёт вертикальное выравнивание элемента относительно его родителя, строки текста или ячейки таблицы.
